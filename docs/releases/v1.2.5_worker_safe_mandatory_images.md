# v1.2.5 — Worker-safe finalization + mandatory image evidence

## Problem fixed

Batch workers could leave partially-created row folders containing only `request.json` and `retailer/manifests/` when an exception occurred before final artifact writing. Also, pages with many image candidates could end with no retained images and an empty/weak `vision.md`.

For this product-coding use case, that is not acceptable: product images are mandatory evidence for product identification.

## New behavior

Every row now writes a final artifact contract, even on failure:

```text
data/scraped/<input_id>/
├── request.json
├── scrape_result.json
├── _COMPLETE.json or _FAILED.json
└── retailer/
    ├── source.md
    ├── product_evidence.json
    ├── product_evidence.md
    ├── claims.md
    ├── vision.md
    ├── metadata.json
    ├── quality_report.json
    ├── source_alignment_report.json
    ├── evidence_recovery_report.json
    └── manifests/
        ├── agent_trace.json
        ├── artifact_manifest.json
        ├── image_manifest.json
        └── table_manifest.json
```

## Visual evidence contract

A clean downloaded product-gallery image is mandatory for `artifact_quality=usable/strong`.

| Visual status | Meaning | Quality impact |
|---|---|---|
| `final_product_images_available` | At least one clean product image retained | usable/strong possible |
| `unverified_images_retained` | Image files retained, but not cleanly vision-confirmed | manual review |
| `screenshot_fallback_only` | Page screenshot retained because image download failed | partial/manual review |
| `image_recovery_failed` | Candidates found or stage attempted, but no image retained | insufficient |
| `no_image_candidates` | No image URLs discovered | insufficient |

## New output columns

Batch CSV now includes:

```text
image_required
screenshot_fallback_used
visual_evidence_status
image_failure_reason
```

## Runtime settings

```env
PCA_IMAGE_REQUIRED=true
PCA_IMAGE_KEEP_UNVERIFIED_ON_VISION_FAILURE=true
PCA_SCREENSHOT_FALLBACK_ENABLED=true
PCA_SCREENSHOT_TIMEOUT=25
PCA_SCREENSHOT_FULL_PAGE=false
```

## Design decision

Screenshot fallback is not a replacement for product image download. It is visual rescue evidence only and keeps the artifact auditable instead of empty.
