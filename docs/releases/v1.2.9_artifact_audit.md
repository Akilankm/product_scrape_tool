# v1.2.9 — Artifact Audit Tool

This release adds a post-run audit utility for scraped product artifacts.

## Why

After a batch run, the folder tree can contain rows in different states:

```text
complete and ready
complete but visual evidence failed
failed but finalized
incomplete with only request.json/manifests
missing required files
empty vision.md
```

The audit tool gives a direct report instead of manually inspecting folders.

## Run

```bash
pdm run audit-artifacts \
  --output-root data/scraped \
  --output-csv data/artifact_audit.csv \
  --summary-json data/artifact_audit_summary.json
```

Strict CI-style mode:

```bash
pdm run audit-artifacts \
  --output-root data/scraped \
  --fail-on-not-ready
```

## Checks

The audit flags:

```text
missing _COMPLETE.json / _FAILED.json
missing request.json
missing scrape_result.json
missing product_evidence.json
missing quality_report.json
missing source_alignment_report.json
missing or empty vision.md
missing image_manifest.json
missing artifact_manifest.json
missing clean image evidence
manual-review required artifacts
failed finalized artifacts
```

## Output

CSV fields include:

```text
scrape_id
artifact_status
ready_for_coding
terminal_marker
missing_files
empty_files
vision_md_empty
image_file_count
visual_evidence_status
artifact_quality
requires_manual_review
manual_review_bucket
error
```

This release does not change scraping behavior. It only adds an audit/reporting layer.
