# v1.3.4 — Contract-safe evidence enrichment

This release adds deterministic semantic enrichment to existing scrape artifact files without changing the artifact contract.

## Boundaries

```text
No artifact file rename.
No new mandatory artifact file.
No folder schema change.
No scraping provider change.
No search added.
```

## Existing files enriched

```text
retailer/product_evidence.json
retailer/quality_report.json
retailer/source.md
retailer/product_evidence.md
retailer/claims.md
```

## JSON placement

`product_evidence.json` keeps its top-level shape. Enrichment is placed under the existing flexible `quality` object:

```text
quality.semantic_enrichment.identity_verification
quality.semantic_enrichment.feature_evidence_readiness
quality.semantic_enrichment.coding_readiness
```

`quality_report.json` receives the same `semantic_enrichment` block for easier downstream consumption.

## Claim rows

Existing claim rows are made clearer with fields such as:

```text
normalized_value
coding_relevance
evidence_summary
downstream_action
transferability
conflict_status
missing_reason
decision_note
```

## Feature readiness

The enrichment generates readiness hints for common product-coding evidence areas including brand, manufacturer, EAN/GTIN, product type, age range, piece count, package contents, material, dimensions, battery requirement, warnings, license/character, and assortment or set count.

## Batch behavior

`run_batch_scrape.py` runs this enrichment after artifact creation by default. Use this flag to skip it:

```bash
--skip-semantic-enrichment
```

The goal is to give the product-coding engine clearer use/review guidance while preserving the current artifact contract.
