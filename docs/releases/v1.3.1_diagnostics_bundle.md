# v1.3.1 — Diagnostics Bundle

This release adds post-run diagnostics for batch review.

## Added

```text
page_classification.py
image_diagnostics.py
batch_triage.py
scripts/run_batch_triage.py
```

## New command

```bash
pdm run triage-batch \
  --input-csv data/batch_scrape_output.csv \
  --output-csv data/batch_triage.csv \
  --summary-json data/batch_triage_summary.json
```

## Triage CSV fields

```text
triage_bucket
triage_priority
page_classification_status
page_classification_confidence
page_classification_reasons
image_failure_bucket
image_recovery_action
image_error_buckets
image_attempt_buckets
image_manifest_item_count
screenshot_fallback_file_detected
```

## Purpose

The triage report helps answer:

```text
Which rows are not product detail pages?
Which rows failed image recovery?
Which rows need source or identity review?
Which rows are ready for product coding?
```

This release does not change scraping behavior and does not add search. It only adds diagnostics and reporting.
