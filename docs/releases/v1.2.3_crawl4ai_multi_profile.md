# v1.2.3 — Crawl4AI Multi-Profile Hardening

This release keeps the scraper **Crawl4AI-only** and adds a robust same-URL capture router.

## Added

- Multi-profile Crawl4AI capture sequence:
  - `standard`
  - `load_wait`
  - `full_page_scroll`
  - `expand_common_sections`
  - `extract_gallery_sources`
  - `shadow_iframe`
  - `retry_relaxed`
- Deterministic capture scoring per profile.
- Best-profile selection before LLM product evidence normalization.
- Weak-capture detection that does not trust HTTP 200 alone.
- Optional Crawl4AI browser session support through `.env`:
  - `PCA_SCRAPE_USER_DATA_DIR`
  - `PCA_SCRAPE_COOKIES_FILE`
  - `PCA_SCRAPE_ENABLE_STEALTH`
  - `PCA_SCRAPE_USER_AGENT`
- Batch output columns for capture diagnostics.

## New diagnostics

These are now available in `scrape_result.json`, `metadata.json`, `artifact_manifest.json`, `quality_report.json`, `agent_trace.json`, and batch output CSV:

```text
capture_profile_used
capture_profiles_attempted
capture_profile_scores
capture_score
capture_grade
real_scrape_evidence
weak_capture_reasons
```

## Interpretation

`success=true` means the artifact was created. For business readiness, inspect:

```text
real_scrape_evidence
capture_grade
artifact_quality
requires_manual_review
```

A thin captcha/shell page can now produce an artifact for traceability, but it will not be treated as a rich product scrape.
