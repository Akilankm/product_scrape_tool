# v1.1.6 — Clean image folder + business-readable Markdown

## What changed

- `images/` is now a clean final handoff folder. It keeps only raster product images that receive a rich vision verdict of `RELATED: yes`.
- Non-image responses such as `.html`, `.bin`, CDN error bodies, SVG/site chrome, unsupported image payloads, and uncertain/unverified candidates are not written or are removed during final cleanup.
- Rejected candidates remain auditable in `manifests/image_manifest.json` with download attempts, status, MIME, and rejection reason.
- `vision.md` is now table-first and business-readable.
- `product_evidence.md`, `claims.md`, and `source.md` are now more decision-oriented and tabular.

## Final image-folder contract

```text
retailer/images/
├── 001_<sha>.jpg
├── 002_<sha>.webp
└── ...
```

Only relevant product/packaging/accessory images should remain here after the vision pass.

## Audit contract

All candidates, including removed `.html`/`.bin`/403/non-product items, remain visible in:

```text
retailer/manifests/image_manifest.json
```
