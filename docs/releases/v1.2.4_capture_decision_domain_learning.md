# v1.2.4 — Capture Decision + Domain Profile Learning

## Fixes

- Empty `0KB` HTML/Markdown captures now score `0`, even when an image candidate is present.
- Image candidates alone no longer prove a successful product scrape.
- `real_scrape_evidence=false` can no longer produce `capture_grade=strong`.
- Rich captures with incidental block/challenge text are marked `mixed_capture` / `mixed_capture_needs_review` instead of contradictory `strong` + `real=false`.
- Batch output now includes explicit decision flags:
  - `capture_decision`
  - `is_weak_capture`
  - `is_block_or_challenge`
  - `has_real_scrape_evidence`
  - `capture_decision_bucket`
- Batch mode learns successful Crawl4AI profile preferences per domain during a run and tries the best profile first for subsequent URLs on that domain.

## Policy

No Firecrawl was added. The scraper remains Crawl4AI-only and same-URL-only.
