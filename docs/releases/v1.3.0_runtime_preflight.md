# v1.3.0 — Runtime Preflight Diagnostics

This release adds runtime/environment preflight checks so common setup failures are caught before a long batch run starts.

## New command

```bash
pdm run runtime-preflight \
  --output-root data/scraped \
  --report-json data/runtime_preflight.json
```

To verify the browser installation explicitly:

```bash
pdm run runtime-preflight \
  --output-root data/scraped \
  --check-browser-launch
```

## Checks

The preflight verifies:

```text
crawl4ai importability
playwright importability
httpx importability
Pillow importability
output-root writability
LLM configuration sanity
optional Playwright Chromium launch
```

## Batch integration

`run_batch_scrape.py` now runs lightweight runtime preflight before workers start.

Useful flags:

```bash
--runtime-preflight-json data/runtime_preflight.json
--check-browser-launch
--skip-runtime-preflight
```

If the preflight fails, the batch stops before creating partial row artifacts.

## Common remediations

```bash
pdm install --prod
pdm run playwright install chromium
```

In AzureML, also verify the selected output path is writable and the `PCA_LLM_*` variables are populated when LLM mode is enabled.
